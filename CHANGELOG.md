# Changelog

## 0.5.4 - 2026-07-23

- 修复队列项编辑、删除、上移和下移按钮因界面持续重绘而无响应的问题。
- 编辑队列消息时改为取回到 ChatGPT 输入框；输入框已有内容时先确认是否覆盖。
- 新增“立即执行”按钮，可覆盖当前输入框内容并直接发送，覆盖前会进行确认。
- 自动执行下一条队列任务时，如果输入框存在草稿，会暂停并询问是否覆盖执行。
- 增加写入失败、发送失败和队列状态变化时恢复原草稿的保护。
- 补充队列交互与确认流程的静态回归测试。

## 0.5.3 - 2026-07-23

- 修复扩展或脚本重载后旧队列面板残留，导致编辑、删除、排序按钮点击无反应。
- 队列 UI 增加脚本实例所有权，检测到失效节点时自动重建并重新绑定事件。
- 修复页面刷新后只依赖停止按钮判断状态，无法识别后台仍在运行的会话。
- 复用任务提醒模块恢复的 `running` / `waiting_action` 状态，阻止误完成和提前发送下一条。
- 队列操作失败时在面板中显示错误提示，不再静默无反应。
## 0.5.2 - 2026-07-23

- 修复项目主页切换到正式项目会话时，旧 DOM 会话 ID 覆盖新 URL 的问题。
- 页面切换时保留旧会话 URL，确保未完成队列由正确的后台标签接管。
- 输入框内容变化后立即刷新“加入队列”按钮状态。
- 复用已有会话标签时防止 Chrome 内存节省挂起；结束后恢复默认状态，不静音、不关闭用户标签。
- 增加项目会话路由、后台队列接管、完成清理和空闲禁入回归测试。

## 0.2.0 - 2026-07-23

- 修复最小化窗口无法判定任务完成。
- 任务存储从单一 `tabId` 升级为多标签绑定。
- 同一会话的运行中任务自动去重。
- 打开任务时复用现有会话标签。
- 避免页面初次加载历史消息时创建重复任务。
- 后台监控页增加周期刷新。
- 自动迁移 v0.1.0 的本地任务数据。

## 0.1.0 - 2026-07-23

- 初始版本。
