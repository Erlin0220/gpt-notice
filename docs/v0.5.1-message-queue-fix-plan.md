# v0.5.1 消息队列修复计划

## P0

- [x] 项目会话显示并恢复消息队列入口。
- [x] 队列按真实会话隔离；项目草稿会话使用标签级临时 Key。
- [x] 仅允许临时会话迁移到正式会话，禁止会话 A 的队列迁移到会话 B。
- [x] 返回、切换会话或关闭页面后，由后台标签接管未完成队列。
- [x] 没有正在执行、等待确认或队列执行中的会话时，禁止加入队列。
- [x] 任务提醒与队列统一使用停止按钮、忙碌状态、等待确认和稳定时间判断。
- [x] 新增助手消息数量兜底，避免相同回复文本导致队列卡住。

## P1

- [x] 输入框存在草稿时暂停自动发送，避免覆盖用户输入。
- [x] 增加跨标签写入锁和发送租约，减少重复发送与状态覆盖。
- [x] 有未完成队列时禁止后台标签清理；完成后延迟 30 秒清理。
- [x] 自动限制已完成历史数量。
- [x] “清除已完成”只删除已完成记录，不删除失败记录。

## 验证

- 普通会话、项目会话、临时会话转正式会话。
- 返回、切换会话、关闭标签和后台接管。
- 空闲状态禁止加入队列。
- 多标签竞争、草稿保护、重复回复和残留忙碌状态。
- 页面刷新、Service Worker 重启和失败重试。
