# v0.5.3 消息队列生命周期修复

## 问题

1. 扩展更新、脚本重载或页面恢复后，旧的队列面板 DOM 仍存在，但对应事件监听器已经失效，导致编辑、删除、排序等按钮没有反应。
2. 页面刷新时，队列只通过停止按钮、忙碌提示等 DOM 状态判断任务是否正在执行；当这些控件尚未恢复时，会将运行中的会话误判为空闲。

## 修复

- 每个队列脚本实例为 UI 根节点写入唯一 owner；发现 owner 不属于当前实例时删除旧节点并重建。
- 队列按钮使用根节点捕获阶段事件委托，并对异步操作增加异常提示。
- 任务提醒内容脚本暴露只读任务状态桥接，队列读取刷新后恢复的 running / waiting_action 状态。
- 后台任务仍在运行时允许加入队列，但禁止自动发送下一条，也禁止将当前队列项误判完成。

## 验证

- 单元测试：刷新后的 taskRunning 可加入队列、阻止派发、阻止误完成。
- 静态测试：UI owner 与任务状态桥接必须存在。
- Chrome Headless：预置失效旧面板并模拟 PAGE_READY 返回 running，验证旧节点被替换、编辑成功、删除成功及加入队列按钮启用。
