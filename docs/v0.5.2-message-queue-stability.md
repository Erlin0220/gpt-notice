# v0.5.2 消息队列稳定性修复

## 修复

- 项目主页 `/project` 与正式项目会话 `/g/.../c/...` 的路由识别以 URL 为准，避免 SPA 切换期间旧 DOM 会话 ID 污染队列归属。
- 切换会话时不再提前覆盖旧会话 URL，后台观察标签会重新打开正确会话。
- 输入框内容变化后主动刷新队列按钮，减少快速输入后按钮仍禁用的问题。
- 复用已有会话标签时设置 `autoDiscardable: false`，队列结束后恢复；用户标签不会被静音、分组或关闭。

## 测试

- 普通会话、项目主页、项目正式会话 Key 解析。
- 项目草稿队列迁移到正式会话。
- 页面切换后创建静音、不可丢弃的后台观察标签。
- 队列完成 30 秒后清理后台标签。
- 暂停队列不创建后台标签。
- 已有标签复用、丢弃恢复和结束后属性还原。
- 空闲会话不能加入队列。
- Chrome Headless 项目页注入、加入队列、路由迁移和空闲禁入烟雾测试。
